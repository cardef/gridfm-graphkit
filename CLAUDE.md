## Architecture

Everything is **YAML-config-driven and registry-based**. A run flows:

1. `cli.py` loads the YAML into a `NestedNamespace` (`io/param_handler.py`) and merges CLI overrides.
2. Factory functions in `io/param_handler.py` (`load_model`, `get_task`, `get_loss_function`, `load_normalizer`, `get_task_transforms`) look up names from the config in the registries defined in `io/registries.py`:
   `MODELS_REGISTRY`, `TASK_REGISTRY`, `LOSS_REGISTRY`, `NORMALIZERS_REGISTRY`, `TRANSFORM_REGISTRY`, `PHYSICS_DECODER_REGISTRY`, `DATASET_WRAPPER_REGISTRY`.
3. A Lightning `Trainer` with an `MLFlowLogger` runs the task; artifacts (`best_model_state_dict.pt`, `normalizer_stats.pt`) land under `mlruns/<exp_id>/<run_id>/artifacts/`.

Key points about the registry system:

- Registration is decorator-based (`@TASK_REGISTRY.register("PowerFlow")`). Adding a new model/task/loss = write the class, decorate it, reference its name in YAML.
- The same task name (`PowerFlow`, `OptimalPowerFlow`, `StateEstimation`) keys three registries: the task (`tasks/*_task.py`), its data transform (`datasets/task_transforms.py`), and its physics decoder (`models/utils.py`). Keep them in sync when adding a task.
- External plugin packages (e.g. `gridfm_graphkit_ee`) register extra components by being imported via `--plugins`; e.g. `SharedMemoryCacheDataset` for `DATASET_WRAPPER_REGISTRY` is not in this repo.

Main modules:

- `tasks/` — `BaseTask` (`base_task.py`) is a `LightningModule` ABC; concrete tasks wire model + losses + normalizers per task. `*_ac_dc_baseline.py` compute ground-truth AC/DC power-balance metrics.
- `models/` — registered: `GNS_heterogeneous` (`gnn_heterogeneous_gns.py`) and `GRIT` (`grit_transformer.py`, with `grit_layer.py`, RRWP positional encodings).
- `datasets/` — `HeteroGridDatasetDisk` (PyG `Dataset`) processes raw grid data (generated by `gridfm-datakit`) into `processed/*.pt`; `LitGridHeteroDataModule` handles splits, normalizers, and task transforms. Normalizers: `HeteroDataMVANormalizer` (fit on train split, restorable via `--normalizer_stats`) vs `HeteroDataPerSampleMVANormalizer` (always per-sample).
- `training/` — loss zoo (`loss.py`, all in `LOSS_REGISTRY`; physics-informed losses like `LayeredWeightedPhysics`, `PBE`) and callbacks (`SaveBestModelStateDict`).
